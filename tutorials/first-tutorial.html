Tutorial de inicio de la documentación

A continuación podrá ver la respectiva documentación del proyecto de software



Como bien se sabe existen dos carpetas principales, que son el server o servidor, que es la parte del backend donde realizan todas las operaciones de la aplicación, básicamente es la api y la otra carpeta principal se llama client o cliente, que es la parte del front-end, que básicamente es la que realiza peticiones al backend y muestra las respuestas en interfaces gráficas

Para poder hacer el recorrido debes de dar click en las palabras de color morado encerradas en paréntesis ()

Para poder hacer el recorrido completo cada que visites un sitio que te indique el tutorial, tendrás que darle en la opción de Get Started para volver al tutorial y poder seguir con la guía


INICIO DE RECORRIDO DE LA DOCUMENTACIÓN

SERVER-BACKEND

Index Server

En primer lugar tenemos el (index) que básicamente aquí se inicia todo, ya que llama a la app y la pone en ejecución

App Server

En la app se puede visualizar las configuraciones y las rutas que llama para aplicarlas (App)

DB Server

En (DB) podrás encontrar la conexión a la base de datos

Config Server

En (Config) podrás ver la configuración de las variables de entorno

Módulo De La Ruta Archivo Post.Rotes

En este archivo (Post.Rotes) podrás encontrar los métodos de uso de la api

Módulo Del Modelo Archivo Post

En este archivo (Post) podrás encontrar el modelo que tendrá nuestra app, básicamente son las claves y restricciones que éstas tendrán para comunicarnos con la database

Múdulo Lib Archivo cloudinary

En el archivo (Cloudinary) podrás encontrar las conexiones a ese servicio para almacenar los archivos que se creen a la hora de ejecutar la app(Cloudinary es un servicio para poder almacenar archivos)

Módulo Controller Archivo Post.Controllers

En el archivo (Post.Controllers) podrás encontrar las respectivas funciones que se utilizaron para poder hacer las operaciones CRUD

CLIENT/FRON-TEND

Archivo App de la carpeta Src

Archivo posts de la carpeta api de la carpeta Src

Archivo NotFoundPage de la carpeta page de la carpeta Src

En el archivo (NotFoundPage) podrás encontrar la página que retorna que la dirección url no ha sido encontrada si la dirección es incorrecta o no existe

Archivo PostForm de la carpeta page de la carpeta Src

Archivo index de la carpeta page de la carpeta Src
